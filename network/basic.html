网络基础

TCP/IP

  • 协议族 IP HTTP FTP UDP DNS TCP

  • 分层管理

分层 -- -- --
应用层 http http数据
传输层 tcp tcp首部+http数据
网络层 ip ip首部+tcp首部+http数据
链路层 网格 以太网首部+ip首部+tcp首部+http数据

  • IP协议

    作用将数据包传送给对方，重要条件：IP地址和MAC地址

    IP间的通信依靠MAC地址，采用ARP协议（Address Resolution Protocol）, ARP是一种用以解析地址的协议，根据通信方的IP就能发查出对应的Mac地址

  • 可靠的TCP协议

    提供可靠的字节流服务

    1. 字节流服务：将大块数据分割成报文段为单位的数据包进行管理

    2. 可靠性：三次握手。使用TCP标志flag——SYN,ACK

      发送端（发送标有SYN数据包）——> 接收端（收到，发送标有SYN/ACK的数据包）——> 发送端（接受，发送标有ACK数据包）——> 接收端（接受处理响应）

域名解析——DNS服务

提供域名到IP地址之间的解析服务，或逆向IP地址反查域名服务

协议间关系

客户端

| 浏览www.example.com/logo.png DNS解析到相应的IP地址。
| Http协议职责：生成针对目标Web服务器的Http请求报文。
| TCP协议职责：将Http请求报文按序号分割成多个报文段，并可靠的传给对方。

DNS

| IP协议职责：搜索对方地址，一边中转一边传送

路由器

| TCP协议职责：将Http请求收到的多个报文段按序号重组合成请求报文。
| Http协议职责：对web服务器请求的内容处理。（需要服务器上的logo.png资源）

服务器

| 同样利用TCP/IP通信协议回传

URI和RUL

URI（统一资源标识符），URL（统一资源定位符）

URI就是某个协议方案表示的资源定位标示符，协议方案是指访问资源所使用的协议类型名称。

  • Uniform规定统一的格式可方便处理多种不同类型的资源
  • Resource资源定义：“可标识的任何东西”
  • Identifier表示可标识的对象，也称为标识符

http:// user:pass @ www.example.js :80 /example/index.html ? uid=1 # ch1

协议方案名 登录信息认证 服务器地址 端口号 文件路径 查询字符 片段标识符
