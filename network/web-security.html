web安全

安全

http缺点

  • 通信使用明文，内容可能被窃听
  • 不验证通信方的身份，可能遭遇伪装
  • 无法证明报文的完整性，所以有可能已经遭篡改。

加密及完整性

  • 通过SSL或TLS的组合使用加密Http通信内容。

  • 常用MD5和SHA-1等散列值校验方法，以及用来确认文件的数字签名方法

HTTPS = HTTP + 加密 + 认证 + 完整性保护

  • HTTPS是身披SSL外壳的HTTP

  • SSL采用公开密钥加密的加密处理方式，也曾为非对称密钥加密，公钥公开，密钥私有。

  • 加密和解密同用一个密钥的方式称为共享密钥加密，也称为对称密钥加密。

认证

HTTP/1.1使用的认证方式

  • BASIC认证（基本认证）

  • DIGEST认证（摘要认证）

  • SSL客户端认证

  • FormBase认证（基于表单的认证）
