状态模式

状态模式

  • 关键：区分事物内部的状态，事物内部状态的改变往往会带来事物行为的改变

  • 例子及应用：电灯程序：点击电灯开关多种状态，在点击事件中实现太多if-else嵌套，抽离成类每个类实现状态翻转逻辑和点击事件

  • 文件上传，状态机。

  • 优点：

    1. 状态模式定义了状态于行为之间的关系，并将它们封装在一个类里，通过增加新的状态类，很容易增加新的状态和转换。
    2. 避免Context无限膨胀，状态切换的逻辑被分布在状态类中，也去掉Context中原本过多的条件分支
    3. 用对象代替字符串来记录当前的状态，使得状态更加一目了然。
    4. Context中的请求动作和状态类中的行为可以非常容易的独立变化而互不影响。

  • 性能优化

    1. 两种选择管理State对象的创建和销毁，第一种是仅当需要时才创建随后销毁，另一种是一开始就创建，并且始终不销毁。
    2. 我们为每个Context对象都创建了一组State，实际是可以共享的，享元模式。工厂模式（享元模式共享）

const StateFactory = (function(){
    const State = function(){};

    State.prototype.clickhandle1 = function(){
        throw new Error('子类必须重写父类的clickHander1方法')
    };
    State.prototype.clickhandle2 = function(){
        throw new Error('子类必须重写父类的clickHander2方法')
    };

    return function(params) {
        const F = function(){
            this.uploadObj = new Upload();
        };

        F.prototype = new State();

        for(param in params){
            F.prototype[param] = params[param]
        }

        return F;

    }
})();

const SignState = StateFactory({

const PauseState = StateFactory({
