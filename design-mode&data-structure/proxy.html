代理模式

代理模式

  • 应用：

    • 防火墙代理：控制网络资源访问，保护主机不让'坏人'接近

    • 远程代理：为一个对象在不同的地址空间提供局部代表，在java中，远程代理可以是另一个虚拟机的对象。

    • 保护代理：用于对象应该有不同的访问权限的情况

    • 智能引用代理：取代了简单的指针，他在饭给你对象时执行一系列附加操作，比如计算一个对象被引用的个数。

    • 写时复制代理：通常用于肤质一个庞大系统的情况。

  • 在javascript中常用的是虚拟代理和缓存代理

  • 代理的意义：单一职责原则，开放封闭原则

虚拟代理实现图片的预加载

const myImg = (function(){
    let imgNode = document.createlement('img');
    document.body.appendChild(imgNode);
    return {
        setSrc: function(src){
            imgNode.src = src
        }
    }
})();

const ProxyGetImg = (function(){
    let img = new Image();
    img.onload = function(){
        myImg.setSrc(this.src)
    }
    return {
        setSrc: function(src){
           myImg.setSrc('file://c:/loadding.gif');
           img.src = src
        }
    }
})();

缓存代理

/*************** 计算乘积 ****************/
const mult = function(){
    return Array.prototype.reduce(arguments, function(a, b){
        return a*b;
    })
}

/*************** 计算加和 ****************/
const plus = function(){
    return Array.prototype.reduce(arguments, function(a, b){
        return a+b;
    })
}

/*************** 创建缓存代理的工厂 ****************/
const creatProxyFactory = function(fn){
    let cache = {};
    return function(){
        var args = Array.prototype.join.call(arguments, ',');
        if(args in cache){
            return cache[args];
        }
        return cache[args] = fn.apply(this, arguments)
    }
}

const proxyMult = creatProxyFactory(mult),
proxyPlus = creatProxyFactory(plus);
