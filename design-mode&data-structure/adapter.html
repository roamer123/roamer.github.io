适配器模式

适配器的作用主要解决两个实体键的接口不兼容问题。使用适配器模式之后，原本由于借口不兼容而不能工作的两个实体能工作。

  1. 适配器模式主要用来解决两个已有接口之间不匹配的问题，他不考虑这些接口是怎样实现的，也不考虑它们将来可能会如何演化。适配器不需要改变已有的接口，就能使它们协同作用

  2. 装饰者模式和代理模式也不会改变原有对象接口，但装饰者模式的作用是为了给对象增加功能。装饰者模式常常形成一条长的装饰链，而适配者模式通场只包装一次，代理模式是为了控制对象的访问，通常也只包装一次。

  3. 外观模式的作用到是和适配者模式比较相似，有人把外观模式看成一组对象的适配器，但外观模式最显著的特点是定义一个新的接口。

案例： 地图数据来源更新新来源，兼容旧模

const getGuangdongCity = function(){
    const guangdongCity = [
        {
            name: 'shenzhen',
            id: 11,
        },
        {
            name: 'shantou',
            id: 12,
        }
    ];
    return guangdongCity;
};

const render = function(fn){
    console.log('开始渲染广东省地图');
    document.write(JSON.stringfy(fn()))
};

const addressAadapter = function(oldAddressfn){
    const address = {};
    const oldAddress = oldAddressfn();

    for(let i=0, c; c = oldAddress[i]; i++){
        address[c.name] = c.id;
    }

    return function(){
        return address;
    }
}
render(addressAadapter(getGuangdongCity));
