chrome

chrome F12

判定是否存在内存泄漏问题

  • 打开开发者工具，选择memory -> Allocation instrumentation on timeline -> start，

  • 开始在界面进行反复的操作（同一操作，定位该操作问题）

    每次操作我们都可以看见一个深蓝色的柱子，高度就对应对象的大小，蓝色柱子表示在timeline最后拍的快照中还存在，灰色柱子表示这个对象在timeline中生成，但结束前已被回收。 如果同一操作反复进行，发现蓝色柱子分布的排列在timeline中，没有变为灰色，则这个操作存在内存泄漏问题。

使用performance火焰图查看问题

  • 选择Performance,进行record。

  • 开始在界面进行反复的操作（同一操作，定位该操作问题）

    查看jsHeap, Document, Nodes, Listeners, GPU Memory相对应图表现象：

    • 如果图表成规律的增长和降低则无问题。

    • 如果出现阶梯状增长，后续陡降我们需要进一步查看火焰图。

      正常情况下，浏览器是不断的出现MinorGC进行回收的，MinorGC会清理年轻代的内存，Major GC（大型）清理永久代。如果出现阶梯状增长后出现陡降现象，可能出现Major GC。如果频繁出现Major GC那么我们认为是有内存泄漏问题的。

      查看火焰图情况，整个操作的调用链我们都可以在火焰图中可以看出，图中标红的三角是浏览器对我们的提醒，我们可以通过对比火焰图和jsHeap的增长趋势定位到具体函数进一步查看代码定位问题。
